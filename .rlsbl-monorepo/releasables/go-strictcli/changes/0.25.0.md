## 0.25.0

### Fixes

- [go-strictcli] **README corrected.** Removed documentation for deleted APIs (Emit, DataCommand, RegisterHandler, typed-globals) and documented the current Outcome handler contract.
- [go-strictcli] **Deterministic cli-test-coverage verdict.** The `cli-test-coverage` check now derives its verdict from the committed `.strictcli/test-coverage.json` manifest (union with any local shards), so it produces the same result on every machine instead of failing with "no coverage data" where the suite has not been run locally. Coverage recording and the check are anchored to the app's construction-time directory, so tests that change directory still record into the repo and the check reads the app's own state when run from elsewhere.
